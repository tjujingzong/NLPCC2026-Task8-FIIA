# FIIA 攻击测评结果记录

## 测评思路概览

本项目参加 **NLPCC2026-Task8 FIIA（叙实性推理一致性攻击评测）**，选择 **Track B: DeepSeek API** 赛道。

### 核心指标

| 指标 | 含义 | 计算方式 |
|------|------|----------|
| **MIR** | Multi-turn Inconsistency Rate | `1 - max(cT, cF, cU) / 10`，范围 [0, 0.6] |
| **Score_i** | 单条攻击得分 | `max(MIR_attack - MIR_orig, 0)` |
| **AttackScore** | 总攻击得分 | `Σ Score_i`，最大 120（200条×0.6） |

### 攻击流水线

```
数据集加载 → 基线摸底(MIR_orig) → 筛选目标 → LLM生成攻击句
    → validate.py 核验(R1-R5) → attack.py 评估(MIR_attack)
    → 计算Score → 排序筛选Top200 → 最终提交
```

### 关键核验规则 (R1-R5)

- **R1**: ID 合法性与字段完整性，不可重复
- **R2**: 谓词(predicate)必须完整保留在改编句中
- **R3**: 小句完整性，假设句的 LCS 覆盖率 ≥ 0.7
- **R4**: 文本保留度，Levenshtein 相似度 ≥ 0.65
- **R5**: 通顺保持度，相对 fluency score ≥ 0.6

## 攻击策略演进

### V1: 探索阶段（0001-0019，10条）

**策略**: 多策略试探——反问句、让步转折、模糊限定语、信息稀释等。

**结果**: 3/10 有效，AttackScore = 0.9。
- 0020(Score=0.5): 「似乎 + 某种程度 + 尽管…但 + 大概」组合拳
- 0019(Score=0.3): 仅加「似乎」（短文本有效）
- 0004(Score=0.1): 仅加「似乎」

**失败原因**: 反问句无效(Score=0)，单模糊词对长文本无效。

### V2: 策略收敛（0020-0048，24条）

**策略**: 聚焦「让步转折 + 多重模糊限定语」组合拳，LLM 每条生成1个变体。初始生成过激（句尾附加"但在某种程度上…并非如此"），导致 R4 大量失败。经手动修剪多余从句后全部通过。

**结果**: 与V1合并后共 34 条，其中 11/34 有效，AttackScore = 2.6。
- 🏆 0048(Score=0.6，满分!): 「尽管…似乎…且…或许并非…但…却在某种程度上有意无意地…大概…」

### V3: 规模化尝试（0051-0200，150条）

**策略**: 进一步强化提示词（多层让步+分散模糊语），批量生成150条。

**问题**: LLM 过度添加模糊语（每条4-6个），R4 通过率仅 25%（37/150）。113条R4失败需降级修复为最小改动攻击（仅加「似乎」）。

**修复后**: 148/150 通过核验，2条因R2/R3内容问题降级修复。
